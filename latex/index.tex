\doxysection*{Getting Started}


\begin{DoxyEnumerate}
\item First, make sure your system is supported and already has R\+OS installed (see table above)
\item Go ahead and clone this repository into your {\ttfamily catkin\+\_\+ws}. Read up on {\ttfamily catkin\+\_\+ws} by \mbox{[}following this tutorial(\href{http://wiki.ros.org/catkin/Tutorials/create_a_workspace}{\texttt{ http\+://wiki.\+ros.\+org/catkin/\+Tutorials/create\+\_\+a\+\_\+workspace}}).
\item Within your {\ttfamily catkin\+\_\+ws} directory, go ahead and compile the code\+:
\begin{DoxyEnumerate}
\item {\ttfamily catkin\+\_\+make}
\item {\bfseries{you may have to run \href{https://www.poftut.com/how-to-install-and-use-dos2unix-command-in-linux/}{\texttt{ dos2unix}} on the {\ttfamily hfl\+\_\+driver/cfg/\+H\+F\+L.\+cfg} file in order to get it to compile}}
\end{DoxyEnumerate}
\item After a successful compile, add the new H\+FL R\+OS packages to your environment\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{echo "source <path/to/your/catkin\_ws>/devel/setup.bash" >> \string~/.bashrc}
\DoxyCodeLine{source \string~/.bashrc}
\end{DoxyCode}

\item In two separate terminals, run the following commands\+:
\begin{DoxyEnumerate}
\item {\ttfamily roscore}
\item {\ttfamily roslaunch hfl\+\_\+driver hfl110dcu.\+launch}
\end{DoxyEnumerate}
\end{DoxyEnumerate}

\doxysection*{C\+PP static code analysis}

R\+OS also comes with static code analysis support, therefore in order to run it for the hfl\+\_\+driver package, type\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{catkin\_make run\_tests roslint\_<name\_of\_ros\_package>}
\end{DoxyCode}


Where in this case you would put hfl\+\_\+driver or udp\+\_\+com in for $<$name\+\_\+of\+\_\+ros\+\_\+package$>$. This will output the errors and warnings on console. If more info is required see \href{http://wiki.ros.org/roslint}{\texttt{ this}}.
